# Agentao v0.2.10-rc2

Re-cut of `0.2.10-rc1`. `rc1` tagged a commit where
`agentao/__init__.py` still reported `0.2.9`, so the CI
tag-vs-package consistency check rejected it. `rc2` carries the
identical feature set with `__version__` aligned to the tag.

Prerelease on top of `0.2.9`. Two initiatives land together:

1. **`agentao.acp_client` becomes a stable embedding facade** for
   non-interactive runtimes (daemons, workflow engines, schedulers).
2. **`/crystallize` gains an explicit `refine` stage** so skill drafts can
   be strengthened against the bundled `skill-creator` guidance before they
   are saved.

Release intent: **prerelease / TestPyPI path**. Tag `v0.2.10-rc2` and mark
the GitHub release as a pre-release so `.github/workflows/publish-testpypi.yml`
runs instead of `.github/workflows/publish.yml`.

## Highlights

- **Non-interactive `send_prompt` and `prompt_once`** on `ACPManager` — one
  shot prompts are safe to run from daemon contexts that cannot service
  interactive server requests.
- **Structured error taxonomy** (`AcpErrorCode`, `AcpClientError`,
  `AcpRpcError`, `AcpInteractionRequiredError`) so embedders can branch on
  failure category without string-matching logs.
- **`/crystallize refine`** — `suggest -> refine -> create` three-stage
  flow, with persistent drafts under `.agentao/skill-drafts/`.

## What's New

### ACP Client — Embedding Facade

`agentao/acp_client/` is now an embedding-oriented surface:

- `send_prompt(..., interactive=False)` raises
  `AcpInteractionRequiredError` instead of latching into the CLI prompt
  path whenever the server issues a request for user input.
- `prompt_once(...)` is a one-shot entry point intended for workflow
  runtimes. Ephemeral clients it creates live in a separate
  `_ephemeral_clients` map and never appear in durable `get_status()`
  output.
- Concurrency contract is strict single-active-turn per named server:
  `send_prompt` blocks on the per-server lock; `prompt_once` fails fast
  with `AcpClientError(code=SERVER_BUSY)` when a turn is already active.
- Session reuse compares both `cwd` and `mcp_servers`; a mismatch on
  either field starts a fresh session.
- `cancel_turn` wins over a latched `INTERACTION_REQUIRED` error — the
  terminal outcome exposed to the caller is cancellation/timeout.

### ACP Client — Structured Errors

- `AcpErrorCode` enumerates the embedding-facing categories, including
  `HANDSHAKE_FAIL`, `SERVER_NOT_FOUND`, `SERVER_BUSY`, `REQUEST_TIMEOUT`,
  `TRANSPORT_DISCONNECT`, `INTERACTION_REQUIRED`, `PROTOCOL_ERROR`.
- `AcpRpcError` preserves the raw JSON-RPC numeric code on `rpc_code` and
  sets the structured `error_code` to `PROTOCOL_ERROR` by default.
  Handshake-time RPC failures on both the `connect_server()` and
  `prompt_once()` paths additionally set `.code = HANDSHAKE_FAIL`,
  matching the non-RPC handshake branch.
- `AcpInteractionRequiredError.details["method"]` carries the raw server
  request method for diagnostics; it is intentionally not a public
  attribute so the internal method name can change without breaking the
  API.

### Skill Crystallization — Refine Stage

`/crystallize` is now a three-stage flow:

```
/crystallize suggest       # generate draft from the current session
/crystallize refine        # improve draft with skill-creator guidance
/crystallize create [name] # save draft into the skills directory
```

- Drafts persist under `.agentao/skill-drafts/<session-id>.json` via
  `agentao/skills/drafts.py`, so `refine` and `create` can pick up the
  work across turns.
- `suggest` stays lightweight — the session content is passed through a
  single LLM call; `refine` is the heavier pass that pulls in the
  bundled `skill-creator` guidance.
- `MemoryCrystallizer` semantics are unchanged; the skill pipeline
  remains independent from memory crystallization.

## Bug Fixes

- **`stop_all()` now closes ephemeral ACP clients.** In-flight
  `prompt_once()` callers previously blocked until their request timeout
  when the manager was shut down mid-call. Ephemeral slots now receive
  the synthetic transport-closed signal alongside durable clients.
- **`load_skill_draft()` tolerates non-object JSON.** A corrupted draft
  containing `[]` or a bare string no longer crashes
  `/crystallize status|refine|create`.
- **`/crystallize suggest` degrades when the draft directory is not
  writable.** The generated `SKILL.md` is still shown, the save failure
  is surfaced as a warning, and the user is redirected to
  `/crystallize create [name]` instead of aborting.

## Publish Checklist

1. Ensure `agentao/__init__.py` reports `0.2.10-rc2`.
2. Run the smoke path:
   ```bash
   uv run python -m pytest tests/
   uv build
   uv run twine check dist/*
   ```
3. Tag and push:
   ```bash
   git tag v0.2.10-rc2
   git push origin v0.2.10-rc2
   ```
4. Create the GitHub release for `v0.2.10-rc2` and mark it **as a
   pre-release** so `.github/workflows/publish-testpypi.yml` runs.
5. After TestPyPI validation, cut the GA `0.2.10` tag for
   `.github/workflows/publish.yml`.
