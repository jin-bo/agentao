# Agentao 0.2.6 — SQLite memory subsystem

v0.2.6 promotes the 0.2.6-rc1 memory subsystem rewrite to general
availability. **Zero library-code changes between rc1 and final** — the
three commits that landed after `a5ade5d` are CI-only patches that unblock
the publish pipeline. See [🔧 Post-RC hardening](#-post-rc-hardening) for
the short list.

## What's new

The headline change is a full rewrite of how Agentao remembers things. The
old JSON-file memory layer is gone; in its place is a SQLite-backed
subsystem (`agentao.memory.*`) that distinguishes three structured data
types — persistent memories, session summaries, and per-turn recall
candidates — and treats them with different retention, scoring, and
prompt-injection rules.

The second big shift is **how memory enters the model**. Previously the LLM
had `save_memory` / `search_memory` / `delete_memory` tools and was expected
to manage its own memory. Now `save_memory` is the only tool the model
sees: relevant memories are scored by `MemoryRetriever` and injected into a
`<memory-context>` block automatically every turn, while durable facts ride
along in a `<memory-stable>` prefix that's friendly to the prompt cache.
Search, delete, filter, clear, crystallize, and review are all CLI-only
(`/memory ...`) — explicit user actions, never something the model does on
its own.

## Highlights

- **SQLite-backed memory** with two stores: `.agentao/memory.db` (project)
  and `~/.agentao/memory.db` (user-scope, cross-project). Schema v3 with
  `DROP COLUMN` auto-migration for existing databases.
- **Two prompt-injection blocks per turn**: `<memory-stable>` (user-scope
  + structural project types, plus a pre-reserved cross-session summary
  tail so prior sessions still reach the model after a restart) and
  `<memory-context>` (top-k dynamic recall against the current query).
- **Five-factor retriever**: tag / title-Jaccard / tokenized-keyword /
  content-snippet / filepath-hint, with CJK bigram tokenization, light
  Latin normalization (plurals, version prefixes), short-query tag-weight
  dampening, and dedup against the stable block.
- **Conservative crystallization with a review queue.** A rule-based
  extractor scans **raw user messages** (never assistant narration or
  LLM-generated summary text) for `preference` / `constraint` / `decision` /
  `workflow` patterns. Candidates land in a review queue and only enter
  live memory when you approve them via `/memory review approve <id>`.
- **Memory subpackage decoupled from the LLM stack.** `import agentao.memory`
  no longer pulls `openai`, `mcp`, `agentao.tools.*`, or `agentao.llm.*`.
  Cold import: **334 ms → 35 ms (~10×)**. Locked in by subprocess-isolated
  regression tests.
- **Stable block budget eviction prefers newest decisions** when budget is
  tight, so fresh user intent is never crowded out by long-tail history.
  Survivors render in `created_at`-ASC order so the prompt-cache prefix
  stays stable across turns.

## ⚠️ Breaking changes

1. **LLM tool surface narrowed to `save_memory` only.** The historical
   `search_memory` / `delete_memory` LLM tools are gone. Search, delete,
   clear, filter, crystallize, and review are now **CLI-only** — issue
   them as `/memory search ...`, `/memory delete ...`, etc. Recall is
   automatic: the model gets relevant memories injected via
   `<memory-context>` without needing to call any tool.

2. **`MemoryRecord` fields removed**: `pinned`, `ttl_days`, `expires_at`
   were speculative and never had a write path. Schema v3 includes a
   `DROP COLUMN` migration that runs silently on existing
   `.agentao/memory.db` files (SQLite ≥ 3.35.0; older versions skip
   gracefully). Any external code that read these fields needs to be
   updated.

3. **Crystallizer no longer reads session summaries.**
   `MemoryCrystallizer.extract_from_sessions()` and
   `MemoryManager.crystallize_recent_sessions()` are removed; replaced by
   `extract_from_user_messages()` and `crystallize_user_messages()`. The
   extractor now only scans raw user-message text — assistant narration
   that happens to contain the words "I prefer" or "always" cannot
   trigger a false-positive.

4. **`/memory clear` semantics widened.** It now wipes **all** session
   summaries via `clear_all_session_summaries()`, including those from
   prior sessions visible through the cross-session tail. Previously it
   only deleted the current session, leaving prior-session summaries to
   silently resurface. If you want to start a fresh session while
   preserving prior-session recall, use **`/new`** instead — it advances
   the session id without touching old rows.

## 🔧 Post-RC hardening

Three CI-only patches landed between `a5ade5d` (0.2.6-rc1 tag) and the
final tag. None of them touch library code, the memory subsystem, or the
CLI surface — they exist entirely to unblock the publish workflow:

- **`actions/checkout@v5` + `astral-sh/setup-uv@v6`** — the v4/v5 pair
  runs on Node.js 20, which GitHub deprecated on the runner. Bumping to
  the next major of each moves workflows to Node.js 24 and clears the
  deprecation warning.
- **`twine check` no longer passes `--repository`** — that flag is valid
  for `twine upload` but not `twine check`, and `publish-testpypi.yml`
  was failing with exit code 2 on every RC attempt. `twine check` just
  validates dist metadata locally; there's no repository to address.
- **Publish smoke step now activates the built venv** (`source
  .publish-venv/bin/activate`) before checking that `agentao` is on
  `PATH`. Calling `.venv/bin/python` directly does not put the venv's
  `bin/` on `PATH`, so `shutil.which('agentao')` returned `None` and the
  smoke test failed even though the entry-point script was installed
  correctly. Same fix applied to both `publish.yml` and
  `publish-testpypi.yml`.

**Zero library-code changes between 0.2.6-rc1 and 0.2.6.** If you already
tested 0.2.6-rc1 from TestPyPI, the final wheel is functionally identical.

## 🙏 Acknowledgements

Two P2 issues caught by Codex review pre-ship; both fixed in `a5ade5d`
before tagging:

- **`/new` was wiping the just-finished session's summaries** before
  `archive_session()` could promote them to "previous-session" status,
  silently breaking cross-session recall whenever the user started fresh.
- **`Agentao._extract_context_hints` was reading `block["content"]`** on
  list-shaped messages instead of `block["text"]`, so every multimodal
  or tool-use message dropped its file paths and `filepath_hint` scoring
  stopped firing in those sessions.

Both have regression guards in `tests/test_memory_session.py` and
`tests/test_memory_renderer.py` so they can't come back unnoticed.
