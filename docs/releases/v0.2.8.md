# Agentao 0.2.8

`0.2.8` promotes the `0.2.8-rc1` ACP client release to general
availability.

The ACP client subsystem, `/acp` CLI surface, domain-based `web_fetch`
permission rules, and the CLI modular refactor are the same feature set
described in [v0.2.8-rc1](./v0.2.8-rc1.md). The final 0.2.8 cut focuses on
release hygiene:

- package version, tag, and release notes aligned to the GA `0.2.8` line
- `build` and `twine` added to the dev dependency group so the release
  smoke path (`uv run python -m pytest tests/`, `uv build`,
  `uv run twine check dist/*`) is reproducible from a local checkout
- local release-check dependencies documented in `pyproject.toml`
- stale model/version examples updated in release-facing docs

## Release Summary

- **Version:** `0.2.8`
- **Git tag:** `v0.2.8`
- **GitHub release type:** regular release, not pre-release
- **Publish workflow:** `.github/workflows/publish.yml`

## Install

```bash
pip install -U agentao
```

## Maintainer Checklist

1. Ensure `agentao/__init__.py` reports `0.2.8`.
2. Run the smoke path: `uv run python -m pytest tests/ && uv build && uv run twine check dist/*`.
3. Create and push the tag:

```bash
git tag v0.2.8
git push origin v0.2.8
```

4. Create the GitHub release for `v0.2.8`.
5. Leave **Set as a pre-release** unchecked so `.github/workflows/publish.yml`
   publishes to PyPI.
