# Agentao 0.2.10

`0.2.10` promotes the `0.2.10` line to general availability.

The feature set — the `agentao.acp_client` embedding facade for
non-interactive runtimes and the new `/crystallize refine` stage — is
the same set described in [v0.2.10-rc2](./v0.2.10-rc2.md). This GA
release is the first cut that actually ships the feature code.

## Important Caveat About `rc1` and `rc2`

Both `v0.2.10-rc1` and `v0.2.10-rc2` were tagged against commits that
carried only the version bump and release notes — the feature code
they described was still in the working tree and never made it into
the tagged commits. The TestPyPI tarballs for those rc tags are
effectively empty.

**Do not install or depend on `0.2.10-rc1` or `0.2.10-rc2`.** Upgrade
from `0.2.9` directly to `0.2.10`.

The old rc tags are left in place for archaeological purposes; they are
not force-moved.

## Release Summary

- **Version:** `0.2.10`
- **Git tag:** `v0.2.10`
- **GitHub release type:** regular release, not pre-release
- **Publish workflow:** `.github/workflows/publish.yml`

## Install

```bash
pip install -U agentao
```

## Maintainer Checklist

1. Ensure `agentao/__init__.py` reports `0.2.10`.
2. Run the smoke path: `uv run python -m pytest tests/ && uv build && uv run twine check dist/*`.
3. Create and push the tag:

```bash
git tag v0.2.10
git push origin v0.2.10
```

4. Create the GitHub release for `v0.2.10`.
5. Leave **Set as a pre-release** unchecked so `.github/workflows/publish.yml`
   publishes to PyPI.
