# Agentao 0.4.12

An **ACP-surface + internal-cleanup** release on top of 0.4.11. The ACP server
gains G4 `session/update` emissions (permission modes and the task plan) and
replays a recorded `todo_write` as a plan on `session/load`. The rest is an
evidence-backed optimization pass — a packaging slim-down, an integer-exact
token-count fast path, two latent working-directory bugs fixed, and a round of
duplication cleanup. Everything upgrades in place via `pip install -U agentao`;
no public Python API, schema, or config break.

The headline:

- **ACP G4 (PR-1): permission modes and the task plan ride `session/update`.**
  A connected ACP client now sees the active permission mode and the current
  plan, and a reconnecting client reconstructs plan state because `session/load`
  replays the recorded `todo_write` as a `plan` update.
- **Five dead optional extras removed.** `[pdf]` / `[excel]` / `[image]` /
  `[crypto]` / `[google]` had no in-tree consumers; dropping them slims `[full]`
  from 122 to 106 packages and stops dragging grpc/protobuf, pandas/numpy, etc.
  into the bundled install.
- **Two latent cwd bugs fixed.** The CLI now reads *and* writes
  `.agentao/settings.json` against the resolved project root, and
  `CodebaseInvestigatorTool` honors the per-session `working_directory` instead
  of the process cwd.

## Why this release

The ACP work continues the conformance roadmap: G4 surfaces plan/modes/commands
as `session/update` so chat/automation clients (the ACP target audience) can
render and resume agent state. Alongside it, a multi-dimension optimization
review of the runtime, packaging, and CLI/ACP surfaces
(`docs/design/optimization-opportunities-review.md`) found a small set of
real, bounded wins: dead packaging weight, a per-character token loop on the
no-tiktoken path, drift-prone duplication, and — the two that actually bite —
a settings round-trip and a tool that both ignored the frozen
`working_directory`. This release lands the ACP surface plus the low-risk
cleanup and the two bug fixes; the larger refactors stay recorded, not shipped.

## ACP G4 — modes and plan over `session/update`

- The active **permission mode** and the current **task plan** are emitted as
  `session/update` notifications, so the client reflects them without polling.
- On `session/load`, a recorded `todo_write` is replayed as a `plan` update, so
  plan state survives a reconnect.
- The `session/update` envelope construction is centralized in one helper, and
  `schema_version` is now correctly typed as an `int` (not a `str`) on the wire.
- Companion design + conformance docs: `docs/design/acp-server-conformance-review.md`
  and `docs/design/acp-g4-plan-modes-commands.md`.

## Packaging slim-down

`[pdf]` / `[excel]` / `[image]` / `[crypto]` / `[google]` are removed from
`[project.optional-dependencies]` and from `[full]`:

- They had **zero import sites** anywhere in shipped `agentao/` code (verified by
  grepping the actual import names). The Gemini path is OpenAI-compatible and
  never imported `google-genai`.
- `[full]` now closes over `cli + web + i18n + crawl4ai + tokenizer` only — 106
  packages, down from 122, enforced by `tests/test_dependency_split.py` against a
  refreshed `tests/data/full_extras_baseline.txt`. (`Pillow` stays only because
  `crawl4ai` pulls it transitively — which confirms the `image` extra was
  redundant.)
- **Breaking for anyone who installed those extras:** `pip install 'agentao[pdf]'`
  (or `excel` / `image` / `crypto` / `google`) no longer resolves. Install the
  underlying package directly (`pip install pymupdf`, etc.) if a skill needs it.

## Bug fixes

- **`.agentao/settings.json` round-trip.** `_load_settings` and `_save_settings`
  both used a cwd-relative `Path(".agentao")`, so a CLI launched from a
  subdirectory persisted `mode` to a file the factory never reads. Both now use
  `replay.config.settings_path()` bound to the agent's frozen
  `working_directory`.
- **`CodebaseInvestigatorTool` cwd bypass.** It resolved its `directory` argument
  with a raw `Path(...).expanduser()`, ignoring the per-session
  `working_directory` binding every other tool honors. It now routes through the
  base `_resolve_directory`, closing a latent per-session-cwd leak under ACP.

## Internal cleanup (behavior-preserving)

- `_heuristic_token_count` replaces its per-character Python loop with an
  integer-exact vectorized count (`encode("ascii", "ignore")` for the ASCII/CJK
  split), removing the O(chars) loop from the no-tiktoken estimation fallback
  with no change in the counted value.
- The microcompaction and full-compression predicates share a single
  per-iteration token estimate instead of computing it twice.
- `factory.resolve_provider_name()` unifies the `LLM_PROVIDER` read across the
  factory, diagnostics collector, and CLI (the ACP provider resolver keeps its
  own raw read to preserve non-ASCII casefold semantics).
- `split_subcommand()` / `unknown_subcommand()` collapse the duplicated slash-
  command dispatch preamble and footer across the `/mcp`, `/sessions`,
  `/permission`, `/acp`, `/agent`, and `/memory` handlers.
- The pure-Python search fallback and the git-grep / rg fast paths share one
  `_format_match_lines` helper for the "No matches / Found N / cap-100" contract.
- A new ACP `resolve_session()` backs both `require_active_session` and
  `session/cancel`.

## What did **not** change

- The `agentao.host` Python contract and the ACP schema snapshot
  (`docs/schema/host.acp.v1.json`) are unchanged — replay, host, and ACP
  schema-drift checks pass clean.
- Permission semantics, memory, replay storage, plugins, and MCP: untouched.
- Deprecations stay on schedule: `agentao.harness` (→ `agentao.host`), the 8
  legacy `Agentao.__init__` callbacks, and the no-op `ToolRunner` compatibility
  kwargs all still warn / no-op and are removed in **0.5.0**, not before.

## Tests

Full suite green locally (3085 passed) and the slow clean-install /
dependency-closure matrix (`tests/test_dependency_split.py`, 3 passed against the
refreshed 106-package baseline), plus the mypy `--strict` typing gate on
`agentao.host` and the replay + host + ACP schema-drift checks. The change set
also passed a workflow-backed code review and a Codex review pass.

## Upgrade

```bash
pip install -U agentao          # library
pip install -U 'agentao[cli]'   # interactive CLI
pip install -U 'agentao[full]'  # all optional extras
```

No migration is needed from 0.4.11 for the typical install. **Only** if you
relied on `agentao[pdf]` / `[excel]` / `[image]` / `[crypto]` / `[google]`:
those extras are gone — install the underlying package directly, or pin
`agentao==0.4.11` until you do.

## Out of scope (deferred)

- The Tier-3 refactors recorded in the optimization review (`_execute_one`
  decomposition, moving `get_conversation_summary` out of core, the
  overflow-recovery nesting) — real but costly, not near-term.
- Remaining ACP conformance gaps beyond G4.
- The 0.5.0 deprecation removals.
