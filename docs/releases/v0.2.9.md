# Agentao v0.2.9

Small GA follow-up to `0.2.8`. No rc cycle — three independent, self-contained
changes on top of the ACP client subsystem and the default-model rollout.

## Highlights

- **Explicit `@server` routing** lets the main CLI delegate to a configured ACP
  server with a single prefix, without a detour through `/acp send`.
- **UTF-8 stdio is now enforced** at the ACP entry point so JSON-RPC can't be
  silently corrupted by a non-UTF-8 default encoding.
- **Default-model messaging matches the runtime again** — wizard, `.env.example`,
  and both READMEs all agree on `gpt-5.4` / `gemini-flash-latest` /
  `claude-sonnet-4-6` / `deepseek-chat`.

## What's New

### ACP Client — Explicit Routing

`agentao/acp_client/router.py` + `agentao/cli/app.py::_try_acp_explicit_route`.

Recognized forms from the main CLI input:

- `@server-name <task>`
- `server-name: <task>`
- `让 server-name <task>` / `请 server-name <task>`

Name matching is longest-first so overlapping names (`qa` vs `qa.bot`) bind
correctly. High-confidence shapes (`@…`, `让 …`, `请 …`) consume the turn even
when `.agentao/acp.json` is missing or broken — delegation intent never silently
falls back to the main agent. Ambiguous colon-prefix shapes fall through, so
ordinary prose like `Note: …` or `url: …` is never hijacked. The ACP config is
re-stat'd by mtime on each attempt, so new/renamed servers mid-session are
picked up without a CLI restart.

`run_acp_prompt_inline()` is extracted from `_acp_send` as the shared runner
used by both `/acp send` and the new routing path.

Part B (experimental `task_complete` push delegation) was evaluated and
dropped: `task_complete` is not part of the ACP standard `sessionUpdate` enum,
so shipping it would require a private server extension. Rationale is in
`docs/implementation/acp-client-project-servers/issues/12-*.md`.

### ACP Client — Env Var Expansion

`AcpServerConfig.from_dict` now expands `$VAR` / `${VAR}` in `env` values
against `os.environ`. API keys and tokens can live in `.env` or the shell
environment instead of being pasted into `.agentao/acp.json`.

## Bug Fixes

- **Force UTF-8 stdio in the ACP server** (`agentao/acp/__main__.py`) — ACP runs
  JSON-RPC over stdio, so a non-UTF-8 default encoding silently corrupts the
  protocol stream. The entry point now reconfigures stdin/stdout/stderr with
  `errors="strict"` at the earliest point, verifies the result, and exits with
  a diagnostic on stderr if the streams cannot be made safe.
- **Default-model messaging aligned with the runtime** across
  `agentao/cli/entrypoints.py` (init wizard), `agentao/llm/client.py`
  (unknown-provider fallback), `.env.example`, `README.md`, and `README.zh.md`.
  `LLMClient._PROVIDER_DEFAULT_MODELS` remains the canonical source.

## Publish Checklist

1. Ensure `agentao/__init__.py` reports `0.2.9`.
2. Run the smoke path:
   ```bash
   uv run python -m pytest tests/
   uv build
   uv run twine check dist/*
   ```
3. Tag and push:
   ```bash
   git tag v0.2.9
   git push origin v0.2.9
   ```
4. The `.github/workflows/publish.yml` workflow runs on the tag push and publishes to PyPI.
5. Create the GitHub release for `v0.2.9` using the CHANGELOG entry as the body.
