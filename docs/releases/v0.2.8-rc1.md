# Agentao v0.2.8-rc1

## Highlights

**ACP Client for project-local server management** — Agentao can now act as an ACP client, connecting to and managing external ACP-compatible agent processes. Configure servers in `.agentao/acp.json` and control them via `/acp` commands.

**CLI modular refactor** — The monolithic `cli.py` (3,246 lines) is replaced by a `agentao/cli/` package (12 modules), improving maintainability and enabling cleaner `/acp` command integration.

## What's New

### ACP Client Subsystem (~2,400 lines)

- `ACPManager` — top-level facade with lazy initialization on first `/acp` command
- `ACPClient` — per-server JSON-RPC 2.0 client over stdio with NDJSON framing; full `initialize` + `session/new` handshake
- `ACPProcessHandle` — subprocess lifecycle management with stderr ring buffer for diagnostics
- `Inbox` — bounded message queue with idle-point flush; ACP server output stays separate from the main conversation
- `InteractionRegistry` — tracks and resolves pending permission/input requests from servers
- Project-only configuration via `.agentao/acp.json` (no global config)

### `/acp` CLI Commands

```
/acp                          # Overview of all servers
/acp start|stop|restart <name> # Lifecycle management
/acp send <name> <message>    # Send a prompt (auto-connects)
/acp cancel <name>            # Cancel active turn
/acp status <name>            # Detailed status
/acp logs <name> [lines]      # View stderr output
/acp approve|reject <name> <id>  # Resolve permission requests
/acp reply <name> <id> <text>    # Reply to input requests
```

### ACP Extension: `_agentao.cn/ask_user`

Agentao advertises a private extension method for requesting free-form text input from the user. `ACPTransport.ask_user()` is fully implemented with graceful degradation — all failure modes resolve to a sentinel string rather than crashing the turn.

### Domain-Based Permission Rules

`web_fetch` now supports domain-tiered permissions in `PermissionEngine`:

- **Allowlist** (auto-allow): `.github.com`, `.docs.python.org`, `.wikipedia.org`, `r.jina.ai`, `.pypi.org`, `.readthedocs.io`
- **Blocklist** (auto-deny): `localhost`, `127.0.0.1`, `0.0.0.0`, `169.254.169.254`, `.internal`, `.local`, `::1`
- Leading-dot suffix matching (`.github.com` matches `github.com` and `api.github.com`)
- Customizable via `.agentao/permissions.json`

### CLI Modular Refactor

`agentao/cli.py` → `agentao/cli/` package with 12 modules:

| Module | Responsibility |
|--------|---------------|
| `app.py` | Main CLI application loop |
| `commands.py` | Core slash commands |
| `commands_ext.py` | Extended commands (ACP, agents, etc.) |
| `entrypoints.py` | CLI entry points and argument parsing |
| `session.py` | Session management |
| `subcommands.py` | Subcommand routing |
| `transport.py` | CLI transport layer |
| `_globals.py` | Shared global state |
| `_utils.py` | Utility functions |
