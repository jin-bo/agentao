# Agentao 0.4.13

A **feature release** on top of 0.4.12. The headline is `/goal` — long-task
continuation with a time/turn budget — plus a small frontmatter-robustness fix.
Everything upgrades in place via `pip install -U agentao`; no public Python API,
schema, or config break.

The headline:

- **`/goal`: state an objective once, and the CLI keeps driving the agent
  toward it.** A host-owned continuation loop runs turn after turn until the
  agent reports the objective complete or blocked, or a time/turn budget trips
  (with one final wrap-up turn). No more typing "continue".
- **Frontmatter that fails to parse now warns instead of vanishing.** A
  malformed SKILL.md / plugin frontmatter block surfaces a warning rather than
  being silently dropped.

## Why this release

Long-running, autonomous "work until it's done" is a recurring ask, but it is a
**host** responsibility, not a harness feature — baking "goal" into the runtime
would hard-code a product decision (how long? what budget? what does *done*
mean?). 0.4.13 adds `/goal` as a **CLI** feature assembled from primitives the
harness already exposes (drive a turn, inject a per-turn message, inject a
tool), and documents the pattern so other hosts can build the same thing. The
harness itself is unchanged. Design record:
`docs/design/codex-goal-mechanism-review.md` §11.

## `/goal` — long-task continuation

```
/goal <objective> [--for 30m] [--turns 10] [--unbounded]
/goal | /goal show | /goal budget … | /goal pause | /goal resume | /goal edit … | /goal clear
```

- **Two budget axes, first-to-trip-wins.** `--for <duration>` caps accumulated
  *active* wall-clock (`90s` / `30m` / `2h` / `1h30m`; paused time excluded);
  `--turns <n>` caps the number of continuation turns. `--turns` is **not**
  `max_iterations` — that bounds the inner tool-call loop within a single turn,
  while `--turns` bounds the outer number of turns. Defaults (settings
  `goal.{default_max_turns, default_time_budget, enabled}`): `25` turns /
  `120m`, with time sized *above* the turn cap so it guards only wall-clock
  pathology and doesn't shadow the turn cap. `--unbounded` opts out.
- **The agent's only write is a guarded, read-only `update_goal` tool.** It can
  set `complete` / `blocked`, and only while the goal is active — a terminal
  `limit_reached` state stays immutable except by the user. The tool is
  injected for the loop's lifetime and restored/removed afterward; it is
  read-only, so it works in read-only-mode analysis goals.
- **State survives restarts.** The goal persists to `.agentao/goal.json`
  (objective, status, caps, usage). A goal left running by a crashed/killed
  process is resumable with `/goal resume`; a budget-tripped goal is revived by
  re-budgeting.
- **Host-owned loop — not the plugin `Stop` path.** `force_continue` is
  hard-capped at 3 and injects a visible message; a goal is a sustained,
  host-driven loop with its own stop condition. New developer-guide chapter
  **4.8 Orchestration Continuation** (EN + ZH) documents the pattern, with
  `docs/guides/goal.md` as the user how-to and `embed-for-agents.md` §7b as the
  distilled embedder pointer.

## Bug fixes

- **Malformed frontmatter warns instead of silently dropping.** Previously a
  SKILL.md / plugin manifest whose YAML frontmatter failed to parse was
  discarded without a trace; it now emits a warning so the misconfiguration is
  visible. (#108)

## What did **not** change

- The `agentao.host` Python contract and the ACP schema snapshot are unchanged;
  replay, host, and ACP schema-drift checks pass clean.
- The harness has no "goal" concept — `/goal` is assembled entirely host-side
  (CLI) from existing primitives; Layer-1 runtime is untouched.
- Permissions, memory, replay storage, plugins, and MCP: untouched.
- Deprecations stay on schedule: `agentao.harness` (→ `agentao.host`), the 8
  legacy `Agentao.__init__` callbacks, and the no-op `ToolRunner` compatibility
  kwargs all still warn / no-op and are removed in **0.5.0**, not before.

## Tests & review

Full suite green (3184 passed) plus the mypy `--strict` typing gate and the
replay + host + ACP schema-drift checks. The `/goal` change set passed two
independent review passes — a workflow-backed multi-agent review (15 fixes) and
a Codex review loop run to convergence (4 more fixes) — and added 94
goal-focused tests.

## Upgrade

```bash
pip install -U agentao          # library
pip install -U 'agentao[cli]'   # interactive CLI
pip install -U 'agentao[full]'  # all optional extras
```

No migration is needed from 0.4.12.

## Out of scope (deferred)

- Token budgets for goals (time/turn only by decision); a per-turn
  usage-observation primitive would be the future, token-budget-only addition.
- Multiple concurrent goals (one goal at a time) and agent-initiated goal
  creation (goals are user-driven).
- Remaining ACP conformance gaps and the 0.5.0 deprecation removals.
